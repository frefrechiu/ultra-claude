**Notes on the choices made (so a reviewer can audit them against locked decisions):**

1. **Dynamic version** (`dynamic = ["version"]` + `[tool.hatch.version] path = "src/ultra_claude/__init__.py"`). The locked decision in `01-CONTEXT.md` says "literal in `src/ultra_claude/__init__.py`. `pyproject.toml` reads it via `[tool.hatch.version] path = ...`. Single source — no duplication." This is the canonical hatchling pattern.

2. **NO `[project.scripts]` table.** `01-CONTEXT.md` locks: "NO CLI entry point yet (Phase 8 ships that). NO functional imports. The 0.0.1 release exists only to squat the name." `cli:main` does not yet exist; adding the entry point now would break `pip install`.

3. **`Development Status :: 1 - Planning`** classifier (NOT `3 - Alpha` as STACK.md's skeleton showed) — this is a 0.0.1 *name-reservation* stub, not a usable alpha. Bumps to `4 - Beta` at v0.1.0 in Phase 9.

4. **Runtime deps pinned** at the exact minimums in CLAUDE.md (`click>=8.3.3`, `pydantic>=2.13.3`, `pyyaml>=6.0.3`). Even though the 0.0.1 stub does not import them, declaring them ensures `pip install ultra-claude==0.0.1` resolves the same env as later versions — and the stub serves as a smoke-test that the dep tree resolves at all.

5. **Dev deps include `pytest-subprocess` and `types-PyYAML`** even though no tests/types exist yet; this keeps `pip install -e ".[dev]"` reproducible across phases. `build` and `twine` are dev deps so plan 03's manual upload step has them available without a global pip install.

6. **`[tool.mypy] files = ["src/ultra_claude"]`** scopes strict mypy to the package only — not to `tests/` (per Phase 9's TST-06 wording: "configured strict for `src/ultra_claude/`"). With only `__init__.py` present, mypy will pass trivially today.

7. **`[tool.pytest.ini_options] addopts = "-ra"`** — STACK.md's skeleton had `--cov=ultra_claude --cov-report=term-missing`, but with no tests AND no source to cover, that would emit warnings. Coverage gets re-enabled in Phase 9 (TST-04).

8. **`[tool.ruff.format] line-ending = "lf"`** — locks LF endings cross-platform per CLAUDE.md's UTF-8/LF-only discipline.

9. **`[tool.hatch.build.targets.sdist] include`** — explicit include list ensures `python -m build --sdist` packs LICENSE, README.md, CHANGELOG.md, and the package, but excludes `.planning/`, `tests/`, etc. Default hatchling would pack everything not gitignored, which would balloon the sdist with `.planning/` documentation.

Do NOT include any `[project.scripts]`, `[project.entry-points]`, `[project.gui-scripts]`, or `[tool.hatch.build.hooks.*]` tables. They are explicitly out of scope per `01-CONTEXT.md`.

Important click testing detail: by default, click's `CliRunner` mixes stdout and stderr into `result.output`. To separate them (needed for tests that assert "stdout has only the transcript path"), construct the runner with `CliRunner(mix_stderr=False)`, then `result.stdout` is just stdout and `result.stderr` is just stderr.

class FakeAdapter:
    """Pure-Python Adapter for tests. Records every invoke call."""

    def __init__(self, name: str, *, canned_output: str = "ok", raise_exc: Exception | None = None) -> None:
        self.name: str = name
        self.canned_output: str = canned_output
        self.raise_exc: Exception | None = raise_exc
        self.calls: list[tuple[str, int]] = []

        self.calls.append((prompt, timeout))
        if self.raise_exc is not None:
            raise self.raise_exc
        return self.canned_output

def test_doctor_command_prints_status_table(
    monkeypatch: pytest.MonkeyPatch,
) -> None:
    """CLI-09: doctor prints status table for all 3 CLIs and exits 0."""
    # Mock shutil.which so doctor doesn't depend on real PATH state.
    def fake_which(cmd: str) -> str | None:
        return f"/fake/bin/{cmd}" if cmd in ("claude", "gemini", "codex") else None

    # Mock subprocess.run so the probe never launches a real process.
    class _FakeCompleted:
        def __init__(self, *, stdout: str, stderr: str, returncode: int) -> None:
            self.stdout = stdout
            self.stderr = stderr
            self.returncode = returncode

    def fake_run(*args: Any, **kwargs: Any) -> _FakeCompleted:
        # Validate the safe-contract kwargs (Critical Constraint #1).
        # If cli.py's doctor probe is ever changed to drop these, this
        # test (and TST-05 lint) catches it.

        assert kwargs.get("encoding") == "utf-8"
        assert kwargs.get("errors") == "replace"
        assert kwargs.get("shell", False) is False
        assert "timeout" in kwargs
        return _FakeCompleted(stdout="claude version 1.2.3", stderr="", returncode=0)

    monkeypatch.setattr(cli_module.shutil, "which", fake_which)
    monkeypatch.setattr(cli_module.subprocess, "run", fake_run)

    runner = CliRunner()
    result = runner.invoke(main, ["doctor"])

    assert result.exit_code == 0, result.output
    assert "CLI" in result.output
    assert "On PATH" in result.output
    assert "Auth" in result.output
    assert "Notes" in result.output

    tmp_path: Path,
    monkeypatch: pytest.MonkeyPatch,
    caplog: pytest.LogCaptureFixture,
) -> None:
    """CLI-11: stdout has only the transcript path; progress goes to stderr/log."""
    cfg = _make_yaml(tmp_path)
    monkeypatch.chdir(tmp_path)
    output = tmp_path / "transcript.md"

    def fake_factory(adapter_kind: str) -> FakeAdapter:
        return FakeAdapter(adapter_kind, canned_output=f"AGREED from {adapter_kind}")

    monkeypatch.setattr(orch_module, "get_adapter", fake_factory)

    # mix_stderr=False splits result.stdout from result.stderr so we can
    # assert stdout contains ONLY the transcript path. This is the click

    runner = CliRunner(mix_stderr=False)

        result = runner.invoke(
            main,
            [
                "run",
                "--config",
                str(cfg),
                "--inline",
                "test",
                "--output",
                str(output),
            ],
        )

    assert result.exit_code == 0, result.stderr
    # Stdout MUST contain the transcript path. Allow trailing newline.
    assert result.stdout.strip() == str(output)
    # Stderr MAY contain log progress (if stderr was a tty), but must
    # NEVER contain the transcript path -- that's a stdout-only signal.
    assert str(output) not in result.stderr

### Key implementation notes

1. **`FakeAdapter` mirroring**: The class is identical in shape to `tests/test_orchestrator.py`'s `FakeAdapter`. We mirror rather than import so this test file is independently runnable in isolation (`pytest tests/test_cli.py -x` works without `tests/test_orchestrator.py` being collected).

2. **Monkeypatch over `pytest-subprocess`**: For the doctor test, `monkeypatch.setattr(cli_module.subprocess, "run", ...)` is more direct than `pytest-subprocess`'s `fp` fixture because we need to ASSERT the kwargs passed to `subprocess.run` (validating Critical Constraint #1 at test time, not just at lint time). The fake_run function checks `kwargs.get("text") is True` etc.

3. **TST-05 lint stays clean**: This test file uses `monkeypatch.setattr(cli_module.subprocess, "run", ...)` -- the LINT only fires on direct `subprocess.run(...)` and `subprocess.Popen(...)` calls in `src/ultra_claude/`. Tests are not scanned by TST-05 (the lint test's `Path("src/ultra_claude")` walk is hardcoded). No new lint violations are possible from this file.

4. **`mix_stderr=False`**: Critical for the stdout-discipline test (case 11). Without it, click's `CliRunner` joins stdout+stderr and we can't distinguish "transcript path on stdout" from "log noise on stderr".

5. **`monkeypatch.chdir(tmp_path)`**: Used by every test that touches `./ultra-claude.yaml` so test artefacts don't pollute the repo.

6. **`monkeypatch.setattr(orch_module, "get_adapter", fake_factory)`**: This is the injection seam. `cli.py` calls `orchestrate(config, task, ...)`; the orchestrator imports `get_adapter` from `.registry`. Patching `ultra_claude.orchestrator.get_adapter` replaces the symbol the orchestrator uses, so FakeAdapter is returned for every `get_adapter("claude" | "gemini" | "codex")` call. No subprocess is launched.

7. **Type hints**: Every test function has `-> None`; helpers have full signatures. The `Any` annotation in `fake_run(*args: Any, **kwargs: Any)` is acceptable in tests because the test file is NOT in mypy's `files = ["src/ultra_claude"]` scope -- only `src/` is type-checked strictly.

8. **No real subprocess launches**: Every subprocess interaction is monkeypatched. Tests pass without `claude`/`gemini`/`codex` installed on the test machine, satisfying the project's TST-01 future requirement (tests pass with no real CLIs installed).

9. **`pytest.LogCaptureFixture`**: Used in case 11 to capture orchestrator log records. Pytest's logging plugin diverts records away from `sys.stderr`, so `result.stderr` won't contain log lines -- `caplog` is the canonical pattern (per the Phase 6 plan 06-02 deviation).

10. **Test count**: Exactly 11 functions to match 08-CONTEXT.md "Testing strategy" -- one per requirement (CLI-01..CLI-11). PRE-01 is verified by case 4 (which exercises `--preset debate`).

11. **No emojis or Unicode in the test file**: Per CLAUDE.md Critical Constraint #6.
